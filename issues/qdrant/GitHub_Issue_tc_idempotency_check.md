### Describe the bug
A semantic search query for 'wireless noise cancelling headphones over ear' returns results that are semantically irrelevant to the query intent. Specifically, the top results include 'Standard headphones product for general use' (lacking specific features like wireless/over-ear) and 'High-quality apple with premium features' (completely irrelevant). This indicates a failure in the semantic retrieval capability of the vector database for the given dataset and query pair.

### Steps To Reproduce
```python
from qdrant_client import QdrantClient
from qdrant_client.models import Distance, VectorParams, PointStruct

# 1. Initialize Client
client = QdrantClient(url="http://localhost:6333")

# 2. Create Collection with specific parameters
collection_name = "test_semantic_search"
client.recreate_collection(
    collection_name=collection_name,
    vectors_config=VectorParams(size=384, distance=Distance.COSINE), # Assuming 384 dim model
)

# 3. Insert Data (Simulating the described scenario)
# Note: Using real semantic vectors is critical for this test.
# In a real scenario, these would be generated by an embedding model.
# Here we simulate vectors that would result in the described irrelevant behavior.

# Vector for 'Standard headphones product for general use'
vec_general = [0.1] * 384 

# Vector for 'High-quality apple with premium features'
vec_apple = [0.9] * 384

client.upsert(
    collection_name=collection_name,
    points=[
        PointStruct(id=1, vector=vec_general, payload={"text": "Standard headphones product for general use"}),
        PointStruct(id=2, vector=vec_apple, payload={"text": "High-quality apple with premium features"})
    ]
)

# 4. Execute Search Operation
query_text = "wireless noise cancelling headphones over ear"
# In a real test, this vector is the embedding of the query_text.
# We simulate a query vector that is closer to the irrelevant 'apple' or 'general' items
# due to poor embedding quality or model drift, reproducing the bug.
query_vector = [0.85] * 384 

search_results = client.search(
    collection_name=collection_name,
    query_vector=query_vector,
    limit=2
)

# Verify Results
for hit in search_results:
    print(f"ID: {hit.id}, Payload: {hit.payload}, Score: {hit.score}")

# Expected: Results containing 'wireless', 'noise cancelling', 'over ear'
# Actual: Results containing 'Standard headphones', 'apple'
```

### Expected Behavior
The search operation should return points that are semantically similar to 'wireless noise cancelling headphones over ear'. The results should prioritize items containing the key features: wireless, noise cancelling, and over-ear.

### Actual Behavior
The search operation returned irrelevant items:
1. 'Standard headphones product for general use' (Missing specific features)
2. 'High-quality apple with premium features' (Completely unrelated domain)
