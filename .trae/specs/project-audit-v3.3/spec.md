# 项目全量审计与实战评估规范 (v3.3)

## Why
项目已升级至 v3.3，引入了意图-数据闭环模型、MRE 自动化验证以及加固后的文档爬取能力。为了验证这些核心改进是否真正解决了之前的“认知断裂”与“假阳性”问题，需要进行一次全流程的实战运行与客观评估。

## What Changes
- **实战运行**：执行 `main.py` 完整生命周期（目标：Milvus v2.6.12，至少 3 轮迭代）。
- **专项审计**：
    - **链接验证**：核实 Issue 中所有文档链接的有效性。
    - **相关性评估**：评估爬取内容与发现 Bug 之间的语义关联度。
    - **复现验证**：手动/自动校验 MRE 代码的复现成功率。
    - **证据链完整性**：审查从文档到执行再到诊断的闭环逻辑。
- **输出评估报告**：客观总结 v3.3 的优劣势，拒绝谄媚，直面遗留问题。

## Impact
- 影响规范：无。
- 影响代码：无（仅读取运行结果）。

## ADDED Requirements
### Requirement: 客观评估报告 (Audit Report)
报告必须包含：
1. **实战指标**：轮次、Token 消耗、验证通过的 Bug 数、拦截的幻觉 Bug 数。
2. **闭环模型验证**：评估 Ground Truth 注入对消除假阳性的实际贡献。
3. **MRE 质量审查**：代码的可运行性及对 Bug 的复现能力。
4. **证据链深度**：引用官方文档的准确性与相关性。
5. **待改进点与理由**：基于实战痛点的下一步优化建议。

#### Scenario: 审计完成
- **WHEN** 完成 3 轮以上实战并审查所有产出物。
- **THEN** 在根目录产出 `AUDIT_REPORT_V3.3.md`。
