# AI-DB-QC 智能测试台 - 项目现状分析与优化计划

## 概述
- **摘要**：AI-DB-QC (LLM-Enhanced Contract-Driven Vector Database QA Framework) 是一个基于多智能体的向量数据库质量保证框架，通过 LangGraph 实现的 7 智能体协同流，实现对向量数据库的自动化测试、缺陷发现与验证。
- **目的**：为向量数据库（如 Milvus）提供全面的质量保证，通过智能体协作发现潜在缺陷，并生成可复现的 MRE 和标准的 GitHub Issue。
- **目标用户**：数据库开发者、QA 工程师、DevOps 团队。

## 目标
- 分析当前项目结构和功能实现
- 评估已完成的核心模块状态
- 识别待优化的关键领域
- 制定后续改进计划

## 非目标（范围外）
- 重新设计整个架构
- 替换核心技术栈
- 实现新的数据库引擎

## 背景与上下文
- **当前版本**：v3.5.2 (Robust Evidence Edition)
- **开发阶段**：Stage 3: 工业级可靠性加固阶段
- **核心技术栈**：Python 3.11, LangGraph, LangChain, 智谱 GLM-4-Plus/Flash, Docker, Pydantic
- **已完成的核心模块**：
  - Multi-Agent Pipeline：基于 LangGraph 的 7 智能体协同流
  - Contract Engine：L1 API/L2 Runtime/L3 Semantic 三层契约
  - Full Observability：结构化 JSONL 遥测与 Token 追踪
  - Harness Engineering：智能熔断器、自愈节点、语义覆盖监控
  - Reliability Shield：持久化集合池 (Pooling)、原子证据落盘
  - Bug Verifier：自动生成 GitHub Issue 与可运行 MRE

## 功能需求
- **FR-1**：多智能体协同工作流，实现从环境侦察到缺陷验证的完整流程
- **FR-2**：基于三层契约的测试用例生成与执行
- **FR-3**：智能语义去重，减少重复缺陷报告
- **FR-4**：证据链验证，确保缺陷报告有可靠的文档引用
- **FR-5**：MRE 自动生成与验证，确保缺陷可复现
- **FR-6**：实时遥测与状态管理，支持运行时监控与故障恢复

## 非功能需求
- **NFR-1**：系统鲁棒性，能够处理大规模并发缺陷处理时的崩溃问题
- **NFR-2**：Token 成本控制，单次实战 Token 消耗稳定在 20 万左右
- **NFR-3**：可扩展性，支持多种向量数据库的测试
- **NFR-4**：可观测性，提供详细的遥测数据和执行日志

## 约束
- **技术**：Python 3.11+, Docker Desktop
- **依赖**：langgraph, pymilvus, sentence-transformers, anthropic API
- **业务**：需要有效的 Zhipu AI API 密钥

## 假设
- 目标数据库（如 Milvus）已通过 Docker 部署
- 环境中已安装所有必要的依赖包
- 有足够的计算资源和网络带宽

## 验收标准

### AC-1：多智能体工作流完整性
- **给定**：系统已正确配置并启动
- **当**：执行主流水线
- **然后**：所有智能体按预定顺序执行，完成从环境侦察到缺陷验证的完整流程
- **验证**：`programmatic` - 检查日志输出和状态文件

### AC-2：智能语义去重效果
- **给定**：系统检测到多个相似缺陷
- **当**：执行去重算法
- **然后**：相似缺陷被合并，去重率达到 60% 以上
- **验证**：`programmatic` - 检查去重前后的缺陷数量

### AC-3：证据链验证有效性
- **给定**：系统生成缺陷报告
- **当**：执行证据链验证
- **然后**：报告中包含有效的文档引用，无虚假 URL
- **验证**：`human-judgment` - 检查生成的 GitHub Issue 中的证据部分

### AC-4：MRE 验证可靠性
- **给定**：系统生成 MRE 代码
- **当**：执行 MRE 验证
- **然后**：MRE 能够成功复现缺陷，验证状态为 SUCCESS
- **验证**：`programmatic` - 检查 MRE 验证日志

### AC-5：系统鲁棒性
- **给定**：系统在大规模并发场景下运行
- **当**：遇到异常情况
- **然后**：系统能够自动恢复，无崩溃或数据丢失
- **验证**：`programmatic` - 检查系统恢复日志和状态文件
